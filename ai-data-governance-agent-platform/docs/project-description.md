# 项目文案

我构建并主导了一个 AI 驱动的企业级数据治理与合规 Agent 平台，主要面向中大型研发、数据、风控和运营团队在数据资产管理、隐私合规、数据质量巡检、指标口径统一和异常修复中的效率瓶颈。该项目不是简单的问答 Demo，而是一套包含前端可视化大屏、后端任务编排、数据源接入、向量知识库、多 Agent 协同、权限审计和自动化闭环处理的完整 AI 平台。

项目解决的核心问题是：企业内部数据表、报表、接口、埋点、指标和离线任务数量持续增长，数据资产分散在数仓、业务库、BI 系统、日志平台和文档系统中，导致字段含义不清、血缘链路不透明、敏感数据识别不及时、权限申请缺少依据、数据质量问题排查周期长。传统治理方式依赖人工登记和定期巡检，效率低且容易遗漏风险，因此我们引入 AI Agent 作为数据治理的智能执行层。

平台采用 RAG + 多 Agent 调度 + 工具调用的架构。用户提出问题后，调度 Agent 会先识别意图，再分配给不同专业 Agent 执行。隐私识别 Agent 会自动扫描表结构、字段注释、样例数据和历史查询记录，判断手机号、身份证、地址、设备号等敏感字段，并生成脱敏建议；血缘分析 Agent 会解析 SQL、调度 DAG、接口调用和 BI 报表依赖，生成跨系统数据链路图；质量巡检 Agent 会结合历史波动、空值率、重复率、枚举分布和业务规则，发现异常数据；权限审计 Agent 会分析用户角色、查询频率、访问范围和审批记录，识别越权访问风险；修复 Agent 则会生成 SQL 修复建议、数据回刷方案、影响范围说明和工单任务。

在后端实现上，平台接入了数据仓库、MySQL、Redis、日志系统、任务调度平台和内部文档库，并通过统一的工具调用协议暴露给 Agent。前端使用可视化大屏展示数据资产地图、风险热力图、Token 消耗、Agent 执行状态、治理工单和修复进度。对于复杂问题，例如“为什么昨日 GMV 指标和 BI 看板不一致”，系统会自动检索指标定义、SQL 变更、上游任务状态、数据延迟、历史复盘和相关负责人，最终输出原因推断、证据链、影响范围和修复建议。

该项目已在约 60 人的数据研发、后端、测试、运营和风控团队中试点使用，覆盖 12 万余个数据资产对象、300 多条核心指标链路和 40 多个业务系统。由于平台需要持续处理数据字典、SQL 解析、文档检索、日志摘要、多轮推理和 Agent 工具调用，日均 Token 消耗约 3,800 万 Token，高峰期在月度审计、版本发布和大促数据保障期间可达到 6,000 万 Token 以上。为控制成本，我们设计了分层模型策略：字段分类、摘要和规则匹配使用轻量模型，复杂血缘推理、合规判断和修复方案生成使用高推理模型，并通过缓存、向量召回过滤、上下文压缩和任务结果复用减少无效 Token。

平台上线后，数据问题平均定位时间从 2 小时缩短到 30 分钟以内，敏感字段发现率提升约 70%，权限审计效率提升约 65%，核心指标口径排查效率提升约 75%。更重要的是，它让企业沉淀的数据规范、历史工单、SQL 经验和审计规则真正被 AI 调用起来，从“人工查文档、找负责人、看日志”转变为“Agent 主动组织证据并生成治理方案”。后续计划继续扩展自动 SQL Review、数据资产评分、合规报告生成、异常自动回滚和跨部门治理驾驶舱等能力。
